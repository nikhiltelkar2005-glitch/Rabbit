🐰 Rabbit API Documentation

Welcome, Prateeksha & Nikhil!
This document covers the complete Rabbit Backend API. The backend supports Communities, Posts, Comments, Events, AMA, DMs, Notifications, Reports, Leaderboard, with College Silos, Trending, Badges, Flairs, and Media Uploads.

Base URL: http://localhost:5001/api
Auth header (protected routes): Authorization: Bearer <jwt_token>

1. Authentication — /api/auth

MethodEndpointDescription
POST/auth/registerRegister with college email — sends OTP
POST/auth/verify-emailVerify OTP → returns token + user
POST/auth/loginLogin → returns token + user
GET/auth/me ✅Get current logged-in user profile
POST/auth/forgot-passwordSend password reset OTP
POST/auth/reset-passwordReset password with OTP
POST/auth/resend-otpResend verification OTP

2. Communities — /api/communities

MethodEndpointDescription
GET/communities ✅Get all communities in your college
POST/communities ✅Create community { name, description }
POST/communities/:id/join ✅Join a community

3. Posts — /api/posts

MethodEndpointDescription
POST/posts ✅Create post (text, poll, or image — multipart/form-data)
GET/posts/community/:id ✅Get posts ?sort=hot|new|top & ?flair=placement
POST/posts/:id/vote ✅Upvote/downvote { type: "upvote|downvote|none" }
POST/posts/:id/vote-poll ✅Vote on poll option { optionId }

Flairs: placement, exam_help, rant, advice, funny, confession, discussion, question, announcement, lost_found

Image upload: Send multipart/form-data with field image (jpg/png/gif/webp).

4. Comments — /api/comments

MethodEndpointDescription
POST/comments ✅Create comment/reply { content, postId, parentCommentId }
GET/comments/post/:postId ✅Get all comments for a post
POST/comments/:id/vote ✅Upvote/downvote a comment

5. Events — /api/events

MethodEndpointDescription
GET/events ✅List events ?filter=upcoming|past|all & ?tag=tech
POST/events ✅Create event { title, venue, eventDate, clubName, tags, maxAttendees }
GET/events/:id ✅Get single event with attendees
POST/events/:id/rsvp ✅Toggle RSVP (call again to un-RSVP)
PATCH/events/:id/cancel ✅Cancel event (organizer or admin)
GET/events/community/:id ✅Events by community

6. Reports / Admin — /api/reports

MethodEndpointAuthDescription
POST/reportsUserSubmit a report
GET/reportsAdminList reports ?status=pending&targetType=post
PATCH/reports/:idAdminUpdate status (reviewed | action_taken | dismissed)
GET/reports/admin/statsAdminDashboard — report counts + banned users
PATCH/reports/admin/ban/:userIdAdminBan a user
PATCH/reports/admin/unban/:userIdAdminUnban a user
DELETE/reports/admin/post/:postIdAdminDelete any post
DELETE/reports/admin/comment/:commentIdAdminDelete any comment

7. Anonymous DMs — /api/dms

MethodEndpointDescription
GET/dms ✅Inbox — latest message per conversation
POST/dms ✅Send DM { recipientAnonymousName, content, originPostId? }
GET/dms/:anonymousName ✅Full conversation thread (auto-marks as read)

8. AMA (Ask Me Anything) — /api/amas

MethodEndpointDescription
GET/amas ✅List AMAs ?filter=open|closed|all
POST/amas ✅Create AMA { title, hostContext, endsAt, tags }
GET/amas/:id ✅Single AMA + questions sorted by upvotes
POST/amas/:id/ask ✅Ask a question { question }
PATCH/amas/:id/answer/:questionId ✅Answer (host only) { answer }
POST/amas/:id/questions/:questionId/upvote ✅Upvote a question (toggle)
PATCH/amas/:id/close ✅Close AMA early (host or admin)

9. Notifications — /api/notifications

MethodEndpointDescription
GET/notifications ✅Get all + unreadCount
PATCH/notifications/read-all ✅Mark all as read
PATCH/notifications/:id/read ✅Mark one as read
DELETE/notifications/:id ✅Delete a notification

Types: post_reply, comment_reply, post_upvote, badge_earned, event_reminder, ama_answer, dm_received

Poll GET /notifications every 30 seconds to keep the notification bell count updated.

10. Leaderboard — /api/leaderboard

MethodEndpointDescription
GET/leaderboard?limit=10 ✅Top karma earners + your own rank

11. Search — /api/search

GET /search?q=internship — Full-text search across posts and communities in your college.

12. Health Check

GET /health — Returns { "success": true, "message": "🐰 Rabbit API is up and running!" }

💻 Notes for Prateeksha (Frontend)

  • CORS: Requests from localhost:3000 or localhost:5173 work out of the box.
  • Auth: Save token from login. Send as Authorization: Bearer <token> for protected routes.
  • Image uploads: Use multipart/form-data (not JSON) when including an image. Field name = image.
  • Notifications bell: Poll GET /notifications every 30 seconds. Show unreadCount as badge.
  • DMs: Poll GET /dms every 30 seconds or on window focus.
  • Error handling: All errors return { "success": false, "message": "..." } — display message in UI.
  • Anonymity: author on posts/comments is always { anonymousName, badge }. Real email is never exposed.
